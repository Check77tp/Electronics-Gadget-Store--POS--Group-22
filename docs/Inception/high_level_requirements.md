# High-Level Requirements

## Functional Requirements

FR1: The system shall allow a cashier to process sales transactions.  
FR2: The system shall update inventory automatically after each sale.  
FR3: The system shall allow managers to add, edit, and delete products.  
FR4: The system shall generate daily, weekly, and monthly sales reports.  
FR5: The system shall manage user accounts and roles (Admin, Manager, Cashier).  
FR6: The system shall allow product search by name, category, or barcode.  
FR7: The gadget store system should be able to communicate with other systems.  
FR8: The system shall allow administrators to create user accounts for employees.
The system shall allow users to log in using a username and password.
FR9: he system shall support different user roles such as cashier and administrator.
FR10: The system shall allow administrators to disable or remove user accounts.
FR11: The system shall allow administrators to view daily sales reports.
12: The system shall allow administrators to generate reports for weekly or monthly sales.
13: The system shall allow administrators to view inventory reports showing current stock levels.
14: The system shall allow administrators to export reports for business analysis.
15: The system shall allow users to search for products by name or category.
16: The system shall allow users to view product details including price and stock availability.

## Non-Functional Requirements

NFR1: The system shall process transactions within 2 seconds.  
NFR2: The system shall ensure data security through authentication and role-based access control.  
NFR3: The system shall be user-friendly and easy to learn.  
NFR4: The system shall maintain data integrity and prevent data loss.  
NFR5: The system shall be scalable for future expansion.  
NFR7: The system should be able to work both online (to send reports and summaries of gadget stock levels to the cloud) and offline (during network interuption periods - so that gadget sales continue.)
